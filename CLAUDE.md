## Comportamento de Resposta (Economia de Tokens)
- Seja extremamente conciso. Responda apenas o necessário.
- Não peça desculpas ou use frases de cortesia (ex: "Claro, eu posso ajudar").
- Vá direto ao código ou ao comando de terminal.
- Se a tarefa for simples (DTOs, logs), use o modelo Haiku.

## Diretrizes Técnicas
- **Ambiente:** AlmaLinux / Distrobox (Arch Linux).
- **Backend:** ASP.NET Core (C#) ou Spring Boot (Java).
- **Ignorar:** Nunca analise pastas `bin/`, `obj/`, `target/` ou `.git`.
- **Estilo:** Use nomes de variáveis em inglês e padrões Clean Code.

## Execução
- Antes de rodar comandos no terminal, verifique se está no diretório correto.
- Em caso de erro de compilação, leia apenas as últimas 20 linhas do log.
